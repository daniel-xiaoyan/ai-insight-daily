## 一、Prompt设计原则

### 1.1 核心原则：清晰、具体、结构化

**❌ 反面示例**：
```
帮我写一篇文章。
```

**✅ 正面示例**：
```
# 角色
你是一位资深的科技行业分析师，擅长撰写深度行业洞察文章。

# 任务
请撰写一篇关于"AI编程工具市场格局"的分析文章，目标读者是科技投资人。

# 要求
- 字数：2000字左右
- 结构：市场概况 → 领先者分析 → 趋势预测 → 投资建议
- 风格：专业、客观、数据驱动
- 必须包含：市场份额数据、头部公司产品对比、未来3年预测

# 输出格式
使用Markdown格式，包含标题、小标题、列表、表格。
```

### 1.2 五大设计原则

| 原则 | 说明 | 示例 |
|------|------|------|
| **角色设定** | 明确AI的身份和专长 | "你是一位资深医生..." |
| **任务明确** | 具体描述要完成什么 | "请分析这份财报的3个风险点" |
| **约束清晰** | 说明限制条件和要求 | "字数不超过500字" |
| **示例引导** | 提供输入输出示例 | "示例1...示例2..." |
| **格式指定** | 明确输出格式 | "用JSON格式返回" |

## 二、常用Prompt模板

### 2.1 角色扮演模板

```markdown
# 角色
你是[具体角色]，拥有[具体 expertise]。

# 背景
[相关背景信息]

# 任务
[具体任务描述]

# 约束
- [约束1]
- [约束2]
- [约束3]

# 输出格式
[格式要求]
```

### 2.2 分析类任务模板

```markdown
# 分析对象
[待分析的内容/数据]

# 分析维度
1. [维度1]
2. [维度2]
3. [维度3]

# 输出要求
- 每个维度给出具体分析
- 指出关键发现
- 提供可执行的建议

# 格式
使用Markdown列表，关键发现加粗。
```

### 2.3 创意生成模板

```markdown
# 创意主题
[主题描述]

# 目标受众
[受众特征]

# 约束条件
- 风格：[风格要求]
- 数量：[数量要求]
- 避免：[禁忌事项]

# 输出格式
1. [创意1] - [一句话说明]
2. [创意2] - [一句话说明]
3. ...
```

## 三、高级技巧

### 3.1 Few-Shot学习

**原理**：通过示例让AI理解任务模式

**示例**：
```markdown
# 任务
将以下客户反馈分类为：正面、中性、负面

# 示例
反馈："产品很好用，推荐！"
分类：正面

反馈："一般般，没什么特别的"
分类：中性

反馈："质量太差，退货了"
分类：负面

# 待分类
反馈："包装精美，但是物流太慢"
分类：
```

### 3.2 Chain of Thought（思维链）

**原理**：引导AI分步骤思考，提高复杂任务准确率

**示例**：
```markdown
# 任务
解决以下数学问题，请展示推理过程。

# 问题
一个农场有鸡和兔，头共35个，脚共94只。鸡兔各几只？

# 请按以下步骤推理
1. 设定变量
2. 建立方程
3. 求解
4. 验证答案

# 解答
```

### 3.3 Self-Consistency（自一致性）

**原理**：多次采样，选择最一致的答案

**使用方法**：
1. 用相同Prompt询问3-5次
2. 比较各次输出
3. 选择出现频率最高的答案
4. 或让AI自己判断哪个答案最好

### 3.4 角色叠加法

**原理**：多个角色协作，提升输出质量

**示例**：
```markdown
# 第一轮：创作
你是创意作家，请写一篇关于AI的短篇小说。

[生成内容]

# 第二轮：编辑
你是资深编辑，请对上文进行润色，提升文笔。

[编辑内容]

# 第三轮：校对
你是校对专家，请检查语法和逻辑错误。

[最终内容]
```

## 四、场景化Prompt库

### 4.1 文案写作

```markdown
# 角色
你是一位资深营销文案专家。

# 任务
为[产品名称]撰写[类型]文案。

# 产品信息
- 产品：[产品描述]
- 目标用户：[用户画像]
- 核心卖点：[卖点列表]
- 调性：[品牌调性]

# 要求
- 字数：[字数要求]
- 风格：[风格要求]
- 必须包含：[必含元素]

# 输出
直接输出文案，不需要解释。
```

### 4.2 代码辅助

```markdown
# 角色
你是一位资深[语言]开发工程师。

# 任务
[具体任务：解释代码/找bug/优化性能等]

# 代码
```[代码]```

# 要求
- [具体要求]
- 添加必要注释
- 解释关键逻辑
```

### 4.3 数据分析

```markdown
# 角色
你是一位数据分析师。

# 数据
[数据内容]

# 任务
分析以上数据，找出[具体问题]。

# 输出要求
1. 关键发现（3-5点）
2. 数据支撑
3. 可视化建议
4. 行动建议
```

### 4.4 学习辅导

```markdown
# 角色
你是一位耐心的[学科]老师。

# 学生水平
[水平描述]

# 任务
解释[概念]，要求：
- 通俗易懂
- 举例说明
- 循序渐进
- 检查理解

# 输出
用对话式语气，分步骤讲解。
```

## 五、常见错误与修正

### 5.1 错误1：过于模糊

**❌ 错误示例**：
```
写个好点的邮件。
```

**✅ 修正**：
```
# 角色
你是一位商务沟通专家。

# 任务
撰写一封跟进邮件，主题：项目进度询问。

# 背景
- 收件人：客户项目经理
- 关系：已合作3个月
- 上次沟通：1周前
- 目的：了解需求文档进展

# 要求
- 语气：专业但友好
- 长度：100字以内
- 必须：提供协助意愿
```

### 5.2 错误2：约束过多

**❌ 错误示例**：
```
写一个100字的段落，必须包含10个关键词，用被动语态，不要用"的"字，要押韵...
```

**✅ 修正**：
```
写一个100字的段落，核心要求：
1. 包含关键词A、B、C
2. 客观陈述语气
```

### 5.3 错误3：没有示例

**❌ 错误示例**：
```
把以下文本改写得更专业。
[文本]
```

**✅ 修正**：
```
把以下文本改写得更专业。

# 改写示例
原文："我觉得这个产品挺好的"
改写："经评估，该产品性能指标符合预期"

# 待改写文本
[文本]
```

## 六、调试与优化

### 6.1 Prompt调试流程

```
1. 写初版Prompt
2. 测试3-5次
3. 分析Bad Case
4. 调整Prompt
5. 再次测试
6. 重复直到满意
```

### 6.2 优化检查清单

- [ ] 角色是否清晰？
- [ ] 任务是否具体？
- [ ] 约束是否完整？
- [ ] 格式是否明确？
- [ ] 是否有示例？
- [ ] 是否测试了边界情况？

### 6.3 A/B测试方法

1. 准备2-3个不同版本的Prompt
2. 用相同输入测试
3. 评估输出质量
4. 选择效果最好的版本
5. 记录最佳实践

## 🤖 熏儿总结

**好的Prompt = 清晰的指令 + 具体的约束 + 恰当的示例**

Prompt工程不是魔法，是清晰的沟通。把AI当作一个聪明但缺乏上下文的实习生，用清晰、具体、结构化的方式告诉它要做什么。

**记住**：
- 没有万能Prompt，需要针对场景调优
- 多测试、多迭代、多记录最佳实践
- 保持更新，模型在进化，Prompt也要进化
