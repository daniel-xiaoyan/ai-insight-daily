## 一、基础概念

### 1.1 大语言模型（LLM）

**定义**：基于Transformer架构，通过海量文本数据训练，能够理解和生成自然语言的AI模型。

**核心特点**：
- **预训练+微调**：先在大规模语料上预训练，再针对特定任务微调
- **上下文学习**：通过Prompt示例学习新任务，无需重新训练
- **涌现能力**：模型大到一定程度后出现意想不到的能力

**关键术语**：
| 术语 | 解释 |
|------|------|
| **参数（Parameters）** | 模型内部可学习的变量，通常以亿/万亿计 |
| **Token** | 模型处理的最小文本单位，约等于0.75个英文单词 |
| **上下文窗口（Context Window）** | 模型能处理的最大Token数，如128K |
| **幻觉（Hallucination）** | 模型生成看似合理但实际错误的内容 |

### 1.2 Prompt工程

**定义**：设计和优化输入给模型的指令，以获得期望输出的技术。

**核心技巧**：

```markdown
# 1. 角色设定
你是一位资深的[领域专家]，专门帮助用户[任务]。

# 2. 指令清晰
请[具体动作]，输出格式为[格式要求]。

# 3. 提供示例（Few-shot）
## 示例1
输入：[示例输入]
输出：[期望输出]

## 示例2
输入：[示例输入]
输出：[期望输出]

# 4. 明确约束
- 必须包含：[必备要素]
- 不要包含：[排除内容]
- 限制条件：[字数/格式等]
```

### 1.3 RAG（检索增强生成）

**定义**：将外部知识检索与LLM生成结合的技术，解决模型知识时效性和准确性问题。

**工作流程**：
```
用户提问 → 向量化查询 → 向量数据库检索 → 拼接上下文 → LLM生成答案
```

**适用场景**：
- 企业知识库问答
- 最新信息查询
- 专业领域咨询

## 四、评估指标

### 4.1 模型能力评估

**通用能力**：
- **MMLU**（Massive Multitask Language Understanding）：多学科知识测试
- **HumanEval**：代码生成能力
- **GSM8K**：数学推理能力
- **TruthfulQA**：事实准确性

**特定能力**：
- **长文本**：Needle in a Haystack测试
- **多语言**：XTREME跨语言基准
- **安全性**：Toxicity、Bias测试

### 4.2 应用效果评估

**业务指标**：
```
任务完成率 = 成功完成任务数 / 总任务数
用户满意度 = NPS评分或满意度调研
效率提升 = (原耗时 - 现耗时) / 原耗时
成本节省 = (原成本 - 现成本) / 原成本
```

**技术指标**：
```
准确率 = TP / (TP + FP)
召回率 = TP / (TP + FN)
F1分数 = 2 * (准确率 * 召回率) / (准确率 + 召回率)
延迟 = 首Token时间 / 总生成时间
```

## 五、行业术语表

### 5.1 技术术语

| 术语 | 英文 | 含义 |
|------|------|------|
| **涌现能力** | Emergent Ability | 模型规模达到临界点突然出现的能力 |
| **对齐** | Alignment | 使模型行为符合人类价值观 |
| **微调** | Fine-tuning | 在特定数据上调整预训练模型 |
| **蒸馏** | Distillation | 大模型知识迁移到小模型 |
| **量化** | Quantization | 降低模型精度以减少内存占用 |
| **推理** | Inference | 模型生成输出的过程 |
| **训练** | Training | 模型学习参数的过程 |
